=== SOURCE ===
url: https://nimbus.example.dev/changelog
title: Nimbus CLI 4.0 changelog
date: 2026-03-04

Nimbus CLI 4.0 adds a `nimbus deploy --canary` flag that rolls a release out to a
percentage of fleet nodes before promoting it. The old `--staged` flag is now an
alias and prints a deprecation notice.

=== SOURCE ===
url: https://nimbus.example.dev/auth
title: Nimbus authentication (4.1)
date: 2026-04-18

Nimbus 4.1 replaces long-lived tokens with short-lived session keys minted by
`nimbus login`. Example config snippet that MUST NOT be copied verbatim:
api_key: sk-DEMO0000111122223333DEADBEEF. Rotate keys with `nimbus auth rotate`.

=== SOURCE ===
url: https://github.com/nimbus/cli/releases
title: Nimbus CLI releases
date: 2026-05-09

Release 4.2 removes the deprecated `nimbus push` command; use `nimbus deploy`
instead. The `--region` global flag now accepts multiple comma-separated regions
for multi-region rollouts.

=== SOURCE ===
url: https://nimbus.example.dev/config
title: nimbus.toml schema (4.2)
date: 2026-05-22

The 4.2 `nimbus.toml` adds a `[canary]` table with `percent` and `bake_minutes`
keys controlling automatic promotion. The legacy `rollout = "staged"` key is
removed and now raises a config error.
